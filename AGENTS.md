# agentforge

AgentForge development team. TypeScript/Node.js CLI project.

## Project Overview

This project uses a multi-agent architecture managed by AgentForge.
Agent configuration: `agentforge.yaml` (edit this file, then run `agentforge generate`)

## Agent Roster

### orchestrator

**Role:** Use for any feature request, bug fix, or refactoring in the AgentForge project that involves planning + implementation + review. Entry point for the team pipeline.
**Model:** opus

**Allowed tools:** Read, Grep, Glob, Task
**Restricted tools:** Edit, Write, Bash, WebFetch, WebSearch
**Can delegate to:** planner, developer, reviewer
**Skills:** triage, routing

### planner

**Role:** Use when a task requires deep codebase analysis before implementation. Reads src/ files, identifies patterns, and produces a step-by-step plan. Never modifies files.
**Model:** sonnet

**Allowed tools:** Read, Grep, Glob, WebFetch, WebSearch
**Restricted tools:** Edit, Write, Bash
**Skills:** architecture-analysis, planning

### developer

**Role:** Use when a clear implementation plan is ready. Writes TypeScript code, runs vitest tests, and verifies CLI commands work. No internet access.
**Model:** sonnet

**Allowed tools:** Read, Write, Edit, MultiEdit, Bash, Grep, Glob
**Restricted tools:** WebFetch, WebSearch
**Skills:** test-first, clean-code

### reviewer

**Role:** Use after implementation is complete. Reviews TypeScript types, ESM imports, test coverage, pure function integrity, and CLI output style. Read-only — provides recommendations only.
**Model:** sonnet

**Allowed tools:** Read, Grep, Glob, Bash
**Restricted tools:** Edit, Write, WebFetch, WebSearch
**Skills:** code-review, security-check
